05 / Strategy

50% Data for Fast Development

Accelerating iteration with a strategic data subset

⚡

Why 50% First?

Training on the full BraTS 2024 dataset is computationally expensive. Using a 50% stratified subset cuts initial training time by ~2× while preserving tumor-class distribution — enabling rapid prototyping and architecture search.

🎯

Stratified Sampling

Cases are selected using stratified random sampling across tumor grades (LGG / HGG) and MRI acquisition sites to ensure the 50% subset is statistically representative of the full dataset distribution.

🔬

Validation Protocol

The held-out 50% acts as a robust external validation set. Once the model exceeds Dice ≥ 0.80 on the subset, full-dataset fine-tuning is performed to push beyond the 0.85 target.

📈

Learning Curve Analysis

We will plot Dice score vs. training set size (10% → 25% → 50% → 100%) to quantify diminishing returns and justify compute trade-offs for resource-constrained settings.

📊 Expected Outcome of 50% Strategy

Reach Dice ≈ 0.82 on 50% data in ~40% of full-training compute → iterate on XAI and LLM components in parallel → final full-data fine-tune achieves target Dice ≥ 0.85. This two-stage approach is aligned with best practices in data-efficient medical image segmentation (Isensee et al., 2021).
